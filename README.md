# NeuroIndex

A premium, cinematic website for NeuroIndex — built with TanStack Start, React 19, Tailwind CSS v4, and Nitro.

## Deploying to Vercel

This project is configured for Vercel via the `nitro: { preset: "vercel" }` setting in `vite.config.ts`. When you build outside the Lovable sandbox, Nitro produces a native Vercel output bundle in `.vercel/output/`.

### From GitHub (recommended)

1. Push this repository to GitHub.
2. Import the project in Vercel: **Add New Project → Import Git Repository**.
3. Vercel should detect the `.vercel/output/` build format automatically. If prompted for framework, choose **Other**.
4. Configure the build settings:
   - Build command: `bun run build` (or `npm run build` / `pnpm run build`)
   - Output directory: leave empty or set to `.vercel/output` (Nitro writes the Vercel bundle here)
   - Install command: `bun install` (or `npm install` / `pnpm install`)
5. Add required environment variables in Vercel:
   - `LOVABLE_API_KEY` — your Lovable workspace API key
   - `GOOGLE_SHEETS_API_KEY` — the Google Sheets connector API key for the inquiry form
6. Deploy. Vercel will build the site on every push to the default branch.

### From the Vercel CLI

```bash
# Install Vercel CLI if you haven't already
npm i -g vercel

# Login and deploy
vercel
```

During `vercel` setup, choose the default options. Because the build produces `.vercel/output/`, the CLI will deploy it as a Vercel project.

## Local development

Requires **Bun** (or Node.js 20+ with `npm`/`pnpm`):

```bash
# Install dependencies
bun install

# Start dev server
bun dev
```

The dev server runs on `http://localhost:8080` by default.

## Building for production

```bash
bun run build
```

With the Vercel preset, the production bundle is written to `.vercel/output/`. This folder is ignored by Git; Vercel will regenerate it during deployment.

## Environment variables

The inquiry form submits to a Google Sheet via the Lovable connector gateway. Set these variables in your hosting provider:

| Variable | Description |
| --- | --- |
| `LOVABLE_API_KEY` | Lovable project API key for authenticating connector requests |
| `GOOGLE_SHEETS_API_KEY` | Google Sheets connector key for the linked spreadsheet |

## Project structure

- `src/routes/` — TanStack Start routes (`__root.tsx` for the root layout, `index.tsx` for the homepage)
- `src/lib/inquiry.functions.ts` — Server function that appends inquiries to Google Sheets
- `src/styles.css` — Global design system, glassmorphism, and motion utilities
- `src/components/ui/` — shadcn/ui components
- `public/` — Static assets (favicon, etc.)

## Notes

- This is a TanStack Start v1 project. Do not create `entry-client.tsx` or `entry-server.tsx`; the Vite plugin handles SSR automatically.
- Do not edit `src/routeTree.gen.ts` directly; it is auto-generated by the TanStack Router plugin.
- Keep the `@tanstack/react-router` and `@tanstack/react-start` imports as-is; mixing with `react-router-dom` will break routing.
- The Lovable sandbox forces a Cloudflare preset during preview builds, but the exported source will use the `vercel` preset when built on Vercel or any non-Lovable environment.
